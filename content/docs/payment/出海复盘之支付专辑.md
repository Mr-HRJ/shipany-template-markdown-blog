# AI出海新手必看！Stripe实战避坑指南

为每个产品单独注册子账号，避免一个产品出问题连累其他项目

✅ **风控设置优化**

✅ **用户体验优化**
在产品中明确展示退订/退款流程和客服联系方式，主动降低纠纷概率

定期检查支付状态，遇到问题及时联系Stripe客服解决

# Stripe 踩的一些坑（英国公司主体）

今天分享一下stripe踩的一些坑

我是英国公司注册的主体，每一笔付款入账有三笔扣款：
1.处理费3.25% + £0.20/笔
2.2%的货币兑换费用
3.Climate：0.1%

仔细研究了一下其实每一笔都有一些应对措施和处理方法：

第一点，应该是每笔都要扣的，无法避免，针对主体不同百分比不一样。我们能做的不做低价，5美金以下的东西压根别去做，扣完款之后，剩下根本没多少，完全就是做慈善

第二点货币转换费：之前跟着教程注册的英国公司和空中云汇，一直以为英国公司主体只能收英镑，中间要多一个货币转换环节，然而今天研究了一下发现居然可以在后台添加美国收款银行账户，直接收USD，直接省了一笔2%的货币兑换费

第三点，这是Stripe的碳中和计划费用，我是为了拿一个高权重外链，不需要完全可以关掉

至于提款，stripe到空中云汇秒到帐，空中云汇到汇丰走FPS也是秒到帐，非常丝滑

很多细节真的只有自己做过了才知道，如果没踩过坑，说明压根就没做过。

# 支付方式信任度的问题

分享一个关于支付方式信任度的问题。

先说结论：除了 Stripe 之外，所有的支付方式都会有信任度问题，会降低转化率。

朋友之前把支付方式从 Stripe 切换到了 Creem，发现转化率明显下降了。

于是他做了一个测试，给所有进入支付页面但取消支付的用户发了一份调查问卷。

结果显示：大部分人不信任 Creem 这个支付方式。

问卷一共有 5 个选项，"信任度"这个原因被设置在最后一个。

所以基本可以排除用户敷衍的问题——如果是随意填写的话，大多数人会默认选第一个选项。

所以如果你的产品面向海外用户，支付环节建议优先选择 Stripe，省下的手续费可能远不及流失订单的损失。
