## 一、site 指定域名搜索：快速确认是否被收录

### 语法

```
site:veo3-3.ai
```

### 我的实际用途

* 检查自己的网站页面是否被搜索引擎收录
* 辅助判断 GSC 中的收录状态是否正常

### 实际结果

* 以 `veo3-3.ai` 为例，通过 `site:veo3-3.ai` 可以看到 **该域名当前被收录了 4 个页面**
* 这说明这个站点**已经进入索引体系**，不是完全的新站或被屏蔽状态

### 使用要点

* `site:` 只能看到**近似数量**，不是精确值
* 但在冷启动或排查收录问题时，已经足够用

---

## 二、inurl 与 allinurl：从 URL 结构找对手

### 语法说明

```
inurl:veo3
allinurl:veo3 generator
```

### 核心区别

* **inurl**：URL 中包含关键词即可（逻辑是“或”）
* **allinurl**：URL 中必须同时包含所有关键词（逻辑是“且”）

### 对比整理

| 语法                        | 逻辑关系 | 实际含义                           |
| ------------------------- | ---- | ------------------------------ |
| inurl:veo3              | 或    | URL 中只要出现 veo3 就会被命中         |
| allinurl:veo3 generator | 且    | URL 中必须同时包含 veo3 和 generator |

### 我的实操思路

* 用 `inurl`：

  * **广撒网**，快速扫一圈相关页面类型
* 用 `allinurl`：

  * **精准过滤**，找和我页面结构高度相似的站点
  * 特别适合分析 **generator、tool、converter** 这类工具站

---

## 三、intitle 与 allintitle：直接定位核心竞争页面

### 语法说明

```
intitle:veo3
allintitle:veo3 generator
```

### 核心区别

* **intitle**：标题中包含关键词即可
* **allintitle**：标题中必须包含所有关键词

### 为什么 title 很关键

* Title 基本决定了页面**主攻关键词**
* 一个页面愿意把词写进 title，说明它**就是冲着这个词来的**

### 对比整理

| 语法                          | 要求     | 适合做什么      |
| --------------------------- | ------ | ---------- |
| intitle:veo3              | 包含即可   | 找泛相关页面     |
| allintitle:veo3 generator | 必须全部包含 | 精准定位直接竞争对手 |

### 关键词难度评估的常用方法

我通常会直接用：

```
allintitle:wan video generator
```

然后看搜索结果数量，来做一个**非常直观的判断**：

* 数量少：

  * 真正围绕这个词做 title 的站点不多
  * **竞争相对可控**
* 数量多：

  * 已经有大量页面正面竞争
  * **新站切入难度明显上升**

这一步，往往比看第三方工具给的“KD 分数”更真实。

## 四、intext 与 allintext：判断正文相关性

### 语法说明

```
intext:veo3
allintext:veo3 generator
```

### 实际含义

* **intext**：正文中出现关键词即可
* **allintext**：正文中必须同时出现所有关键词

### 我的使用场景

* 判断一个页面是不是**顺带提到**这个词
* 还是**正文核心内容就围绕这个词展开**

### 对比整理

| 语法        | 判断对象 | 用途         |
| --------- | ---- | ---------- |
| intext    | 正文   | 判断泛相关      |
| allintext | 正文   | 判断内容是否高度聚焦 |

在分析对手时，我更看重 **title + intext 的组合**，而不是单看某一个指标。

## 五、一个完整的实战分析顺序

如果我现在要评估 **veo3 generator** 这个词值不值得做，通常会按下面顺序来：

1. `site:自己的域名`

   * 确认站点是否正常被收录
2. `allintitle:veo3 generator`

   * 看**正面竞争对手数量**
3. `allinurl:veo3 generator`

   * 看是否有大量**结构相似的工具站**
4. `allintext:veo3 generator`

   * 判断已有内容的**正文深度**
5. 手动点前 10 个结果

   * 看页面质量、内容厚度、是否是纯AI堆词

做到这一步，基本就能判断：
**这个词是“能打”、 “可以绕”、还是“先放一放”。**

## 结论

* Google 搜索语法不是技巧，而是**最低成本的SEO分析工具**
* **allintitle** 是我判断关键词难度时最常用的一招
* **inurl/allinurl** 特别适合分析工具站、生成器站
* 配合人工阅读，比单纯看工具数据更可靠

这些语法本身不复杂，关键在于**你拿它来回答什么问题**。只要问题对了，结论通常也不会差。
