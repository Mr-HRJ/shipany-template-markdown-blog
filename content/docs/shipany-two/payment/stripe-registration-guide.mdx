# 写在前面

这里把 X 上一位同学的分享整理并补充了一下（整体思路和我当时的路径很像，不过我当时用的是中银香港卡）。

如果你打算做出海网站、工具产品、订阅服务，Stripe 是目前主流也最简单集成的收款方式之一。虽然 Creem 对国内用户相对友好，但它有个缺点：**每个网站都会被当作独立产品，每次都要单独审核一遍**，对多项目来说比较麻烦。

很多人以为 Stripe 一定要注册美国公司、拥有 SSN 才能开通。本文分享我自己的实测流程：用**大陆护照 + 众安香港银行卡 + 国内手机号**成功注册 Stripe 香港个人账户，并通过身份验证，进入可收款状态。

# 关于风险

**Stripe 个人账户做出海独立开发，有风险吗？**

结论先说：**能用，但有真实风险，只适合短期验证，不适合长期跑业务。**

Stripe 的个人账户并不等于“随便个人就能长期稳定全球收款”。前期可能较松，但**一旦流水上来，极容易触发 KYC 审核**，要求补充身份、地址、业务说明、网站条款等材料，**解释不清就可能被冻结**。

最严重的情况是：**账户被关闭 + 资金冻结最长 180 天**。期间不能提现、不能周转，广告费、服务器、退款压力全要自己扛，很多独立开发项目会被直接拖死。

**风险更高的场景**包括：

- SaaS / AI 产品
- 订阅制
- 虚拟服务、数字产品（Stripe 对“个人 + 可规模化产品”天然更谨慎）

那为什么还有人用？因为它**适合 MVP 阶段**：

- 月收入低（&lt;1000–2000 美元）
- 只做产品和定价验证
- 能接受被风控的可能性

但**只要你已经稳定赚钱、打算长期做，或依赖这笔钱生活**，继续用个人账户就是在赌运气。

**现实建议很简单：**

- 短期测试：个人 Stripe 可用，但别囤钱
- 长期出海：尽早公司化（香港 / 新加坡 / 美国），稳定性远高于个人账户

一句话总结：**个人 Stripe 只能“试水”，不能“长期航行”。**

# 什么是 Stripe？

Stripe 是一个面向全球开发者和商户的支付平台，可以接受 Visa、Mastercard 等多种国际信用卡，适合 SaaS、出海电商、订阅服务等收款需求。

Stripe 有多个地区站点，很多人选择注册香港或美国站点。

---

# 现在选择 Stripe 的优势？

- 不需要注册海外公司或美国身份
- 支持个人注册（Individual / Sole proprietor）
- 只需护照 + 香港银行卡 + 中国大陆手机号即可开通
- 支持绑定众安、ZA Bank 等香港虚拟银行账户
- 手续费透明、集成方式简单，支持 API 和链接收款

---

# 注册 Stripe 账户所需材料

我的实测成功配置如下：

| 项目 | 我的配置 |
|------|----------|
| 注册地区 | 非大陆地区 IP |
| 身份证件 | 大陆护照 |
| 银行账户 | 众安银行（ZA Bank）个人账户 |
| 电话号码 | 中国大陆 +86 手机号 |
| 认证方式 | 上传护照照片 + 手机人脸识别 |
| 注册邮箱 | Gmail（推荐使用 Gmail / Outlook / 公司邮箱） |

# Stripe 注册流程（个人实测）

## Step 1：注册账号

- 进入官网：https://dashboard.stripe.com/register
- 注册地区选择「中国香港特别行政区」
- 商家类型选择「Individual / Sole proprietor」
- 填写基本信息（建议如实填写，包括护照姓名、出生日期、邮箱等）

> 注意：手机号可直接填写 +86 大陆手机号，亲测可用。

<img src="https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/cd76c5c08d1fd9c3.jpg" alt="image" />

---

## Step 2：填写商家信息

- 行业类型：建议选择“软件/服务类”或“咨询类”等通用类型
- 网站地址：建议填写你正在搭建的网站或你的博客（必须能访问）
- 产品描述：如“提供数字产品订阅服务”、“在线工具收费使用”等
- 客服电话：可填写你的国内手机号
- 账单展示信息：可填写个人品牌名或站点名称

<img src="https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/ec66c348555d4c28.jpg" alt="image" />

---

## Step 3：添加银行账户

- 银行选择众安银行（ZA Bank）
- 登录众安银行 App，复制你的港币账户信息
- 填写银行代码、分行代码、账户号码
- 英文地址信息（可按 App 显示填写）

> Stripe 绑定香港账户后，将以港币结算，可提现至香港卡内。

---

## Step 4：设置双重验证

- 推荐方式：使用 Google Authenticator（身份验证器）App
- 用手机扫码绑定，输入 6 位动态验证码
- 获取并保存备份验证码，防止设备丢失时无法登录

<img src="https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/8719f20dd48d0f59.jpg" alt="image" />

---

## Step 5：身份认证（KYC）

注册完成后，进入：设置 → 商家 → 身份验证，然后按提示：

1. 上传护照照片（护照信息页）
2. 跳转到手机端，进行人脸识别验证  
   - 手机打开 Stripe 提供的验证链接（推荐 Chrome）
   - 跟随引导拍摄并提交

验证完成后，系统会提示认证成功。

<img src="https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/de8a4623378eb0fb.jpg" alt="image" />

### 重要提醒：如果你是大陆护照，别在这里被“香港身份证号码”卡住

有些人会在这一步看到类似下面的页面，要求填写「香港身份证件号码」，示例格式为 `X123456(A)`（你发的截图就是这种）。

<img src="https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/a5bbe13d6201fc0d.jpg" alt="image" />


**结论：大陆护照不能填、也不应该填这个字段。**

出现该页面通常意味着：系统把你走到了「香港居民（HKID）」的验证路径（比如前面选择了 Hong Kong ID / local resident 之类的选项，或流程识别错误）。

正确做法只有一个：

- 不要乱填、不要编造
- 直接中止当前流程
- 用无痕模式/换浏览器/换邮箱重新注册
- 重新走「非香港居民 + Passport（护照）」路径  
  正常情况下会让你上传护照并做人脸识别，全程不会要求 HKID

---

## Step 6：确认激活状态

- 进入 Stripe 后台首页，点击左上角头像 → 设置
- 如果你能看到「API 密钥」中的公钥以 `pk_live_` 开头，说明已进入正式收款模式
- 此时可以正常创建产品、生成收款链接，开始使用
