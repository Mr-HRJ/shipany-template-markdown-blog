---
title: '如何通过收入榜单找到真实赚钱的产品'
---

在做产品或者找需求的时候，我有一个长期坚持的方法：**先看谁在赚钱，再研究他们是怎么赚钱的。**

原因很简单：

* 流量榜单只能说明产品有人访问
* **收入榜单说明产品已经被市场验证**

如果一个产品已经持续产生收入，那么至少证明三件事：

1. 用户愿意为这个需求付钱
2. 产品形态已经被市场接受
3. 商业模式是跑得通的

真正的难点不在于需求是否存在，而在于：**你能不能把产品推广到需要的人面前。**

下面是我常用的几个收入榜单和数据渠道。

---

# Toolify 收入榜单

Toolify 是目前比较常见的 AI 产品数据平台，其中有一个专门的收入榜单，可以直接查看。

访问地址：

[https://toolify.ai/Best-AI-Tools-revenue](https://toolify.ai/Best-AI-Tools-revenue)

![image](https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/a5b680335d1dcc0a.jpg)


这个榜单的特点是：

* 汇总了多个 **常见支付渠道的数据**
* 能看到 **AI 产品的大致收入情况**
* 可以快速找到 **正在赚钱的 AI 工具**

通过这个榜单通常可以观察到：

* 哪些 AI 工具类产品已经有稳定收入
* 哪些细分领域正在增长
* 哪些产品已经形成商业模式

对于做独立开发的人来说，这类榜单最大的价值是：

**快速发现已经跑通的产品方向。**

---

# 通过 Stripe 入站流量推算产品收入

除了直接看收入榜单，我还经常用一个方法：**反向分析支付网关的流量。**

Stripe 是海外最常见的支付系统之一，很多 SaaS 和 AI 工具都在用。

![image](https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/f5b753a532a73a6b.jpg)

操作步骤很简单：

1. 打开 Similarweb
2. 进入网站分析
3. 查看流量来源渠道
4. 输入 Stripe 的网址

这样可以看到：

**哪些网站的用户正在跳转到 Stripe 进行支付。**

虽然这个方法不能直接看到真实收入，因为还会受到两个因素影响：

* 客单价
* 转化率

但它仍然能在一定程度上反映出：

**哪些网站正在产生大量支付行为。**

换句话说：

**如果某个网站持续给 Stripe 导入大量流量，说明它很可能在持续赚钱。**

---

# 通过 Creem 支付网关分析收入来源

除了 Stripe，还可以使用同样的方法分析其他支付网关，比如：

* Creem

方法和 Stripe 基本一致：

1. 使用 Similarweb
2. 查看 Creem 的入站流量
3. 查看流量来源网站

这里有一个很重要的筛选条件：

**选择 referral 来源。**

这样可以过滤掉：

* 搜索引擎
* 社交媒体

留下的基本就是：

**真实的支付跳转来源。**

通过这些来源网站，就可以找到：

* 使用该支付网关的 SaaS 产品
* 正在产生交易行为的工具网站

本质上，这是一种 **通过支付行为反向发现赚钱产品的方法。**

# TrustMMR 实际支付数据平台

TrustMMR 是另一个非常有价值的数据平台。
![image](https://pub-8dc0158e77a140d4b502b52ab75765b5.r2.dev/docs/173458d0388b1542.jpg)

它的核心特点是：

**数据来源是用户主动上传，并且通过 Stripe 支付记录进行验证。**

因此数据的可信度相对比较高。

在 TrustMMR 上可以查看多个关键指标，例如：

* 总收入
* MMR（Monthly Recurring Revenue）
* 增长情况
* 流量
* 单位访客价值

这些数据对于分析 SaaS 产品非常重要。

例如：

| 指标     | 含义          |
| ------ | ----------- |
| 总收入    | 产品累计收入      |
| MMR    | 每月经常性收入     |
| 增长     | 收入增长趋势      |
| 流量     | 网站访问量       |
| 单位访客价值 | 每个访客带来的平均收入 |

其中 **单位访客价值** 是一个非常关键的指标，因为它能反映：

**一个产品的流量变现能力。**

举个例子：

如果两个产品流量一样：

* 产品 A 每个访客价值 0.5 美元
* 产品 B 每个访客价值 5 美元

那么产品 B 的商业模型明显更健康。

# 为什么收入榜单比流量榜单更重要

很多人做产品喜欢盯着流量榜单，但流量本身并不能说明商业价值。

一个更有效的判断逻辑是：

1. **先找赚钱的产品**
2. 再分析它的流量来源
3. 再研究它的产品设计
4. 最后拆解它的商业模式

收入榜单本质上是在帮你回答一个问题：

**市场已经为哪些需求付钱。**

当你找到这些产品之后，真正的工作才开始：

* 分析用户是谁
* 分析他们从哪里来
* 分析产品如何转化
* 分析定价策略

当这些信息逐渐清晰之后，你就能更容易找到：

**真正有商业价值的产品机会。**
