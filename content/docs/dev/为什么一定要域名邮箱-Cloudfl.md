title: '为什么一定要域名邮箱（Cloudflare绑定域名邮箱）'
---

> 这篇文章复盘了我在接入支付和平台审核时，**快速搞定域名邮箱**的一套低成本方案。核心做法是：先把域名的 **NameServer 接入 Cloudflare**，再使用 **Cloudflare Email Routing** 创建**免费的域名邮箱转发**。这套方案几乎零成本、配置速度快，非常适合网站早期用于**支付审核、联系方式展示**等场景。但需要明确的是，它只支持**收件转发，不支持发信**。文章最后也给出了在用户量起来后，升级到可发信方案的几条现实路径。


在接入支付、广告平台或某些第三方服务时，经常会遇到一个硬性要求：

* 联系邮箱必须是 **域名邮箱**
* 不能使用 Gmail、QQ、163 这类公共邮箱

这一步如果卡住，后面的流程基本都走不下去。

问题在于：
**很多人并不是真的需要一个“完整邮箱系统”，只是为了过审核。**

---

# 快速方案的核心思路

我的解决方案是：

* 不自建邮箱服务器
* 不买付费邮箱
* 直接用 **域名邮箱转发**

核心工具只有一个：

* **Cloudflare**

前提条件只有一个：

* **域名的 NameServer 必须接入 Cloudflare**

---

# 一、把域名接入 Cloudflare

如果你之前已经把域名接到 **Vercel**，也不用担心：

* Cloudflare 会自动扫描现有 DNS
* 正常情况下不会影响现有网站解析

## 1. 在 Cloudflare 添加域名

操作顺序如下：

1. 进入 Cloudflare 控制台主页
2. 添加域名
3. 输入你的域名
4. 选择 **免费计划**

## 2. 检查 DNS 记录

Cloudflare 会尝试导入你原有的 DNS 记录。

我的实际情况是：

* 新域名
* 我把原有 DNS 全部清空了

如果你是已经在用的老域名，一定要确认：

* A 记录、CNAME 没被误删
* 网站解析仍然正常

## 3. 绑定 Cloudflare 的域名服务器

这一步是关键。

我的域名注册商是 **Spaceship**，流程是：

1. 进入域名管理后台
2. 找到 NameServer 设置
3. 选择自定义 NameServer
4. 填入 Cloudflare 提供的两条 NS 地址

本质上就是：

**用 Cloudflare 的 NS，替换掉原域名商的 NS。**

## 4. 等待生效

* 我个人的经验：**大概 10 分钟左右**
* 有时也可能更久，取决于 DNS 缓存

Cloudflare 后台会显示生效状态，等变为正常即可。

---

# 二、使用 Cloudflare 邮件路由创建域名邮箱

域名接入完成后，就可以开始设置邮箱了。

## 1. 启用 Email Routing

在 Cloudflare 控制台中：

* 找到“电子邮件”
* 进入 **Email Routing**
* 启用邮件路由功能

## 2. 创建邮箱转发规则

接下来按引导操作即可：

* 输入邮箱前缀

  * 比如：support、contact、admin
* 选择一个**接收转发的真实邮箱**

  * 比如你的 Gmail 或其他常用邮箱

确认后，Cloudflare 会自动完成配置。

## 3. 最终效果

创建完成后：

* `support@yourdomain.com`
* `contact@yourdomain.com`

发到这些地址的邮件，都会**自动转发**到你设置的真实邮箱。

---

# 这个方案的边界和限制

这一点一定要说清楚，避免误用。

## 能做什么

* 用于支付平台、第三方服务审核
* 用作网站“联系方式邮箱”
* 接收用户来信

## 不能做什么

* **不能用这个邮箱发信**
* **不能直接回复用户**
* 不适合作为注册邮件、通知邮件

它本质只是一个**收件转发器**。

---

# 用户量起来后的升级方案

当你的网站进入下一阶段，这个转发方案就会不够用了。

下面是我更现实的升级路径。

## 方案一：发信服务

如果你需要从域名邮箱发信，可以考虑：

* 使用 Cloudflare Workers 结合发信服务
* 或使用 **Resend**

这类服务的特点是：

* 通常每天有 **100 封左右的免费额度**
* 适合早期注册邮件、通知邮件

## 方案二：完整域名邮箱

如果你需要一个“真正能用来沟通”的邮箱：

* 可以考虑 **Zoho** 提供的域名邮箱

适合场景：

* 需要人工回复用户
* 客服沟通
* 长期对外使用

---

# 核心结论

把这套流程压缩成一句话就是：

* **Cloudflare 邮件转发，是最快、最省钱的域名邮箱方案**
* 非常适合网站早期过审核、补联系方式
* 但它不是终局方案

在项目初期，**先跑通流程最重要**；
等用户和业务起来了，再升级到更完整的邮箱体系，才是更合理的节奏。
