在做出海产品时，邮件几乎是不可替代的基础设施。

常见的几个核心场景包括：

* 用户注册后的邮箱验证
* 支付成功后的收据或欢迎邮件
* 密码重置邮件
* Magic Link登录
* 后续的Newsletter或用户召回邮件

如果邮件系统不稳定，这些关键流程都会受到影响。

我最初使用的是 **Supabase自带邮件服务**。测试后发现一个严重问题：

**Gmail邮箱经常完全收不到邮件，甚至垃圾邮箱里也没有。**

原因很简单：

* Supabase免费版使用的是 **共享SMTP**
* 发件域名不是自己的
* Gmail等邮箱服务商会对共享发信信誉做严格过滤

结果就是：**邮件被直接拦截。**

为了提高送达率，我重新搭建了一套邮件系统，核心方案是：

**Resend + 自有域名 + Supabase Custom SMTP**

从注册到上线，总耗时不到一小时。

最终选择 Resend，主要有几个原因。

对开发者来说，这一点非常关键。

# 注册账号与域名验证

首先注册 Resend 账号。

进入：

```
Dashboard → Domains → Add Domain
```

免费版支持 **1个域名**。

我选择使用 **子域名** 作为发信域名：

```
mail.yourdomain.com
```

例如：

```
mail.xxx.net
```

这是邮件系统的最佳实践。

### 为什么使用子域名

原因有两个：

* 不影响主域名DNS配置
* 发信信誉受损时不会影响主域名

邮件服务商对域名信誉有严格评估，如果发信质量不好，域名可能被降权。

使用子域名可以避免风险扩散。

---

# DNS配置

添加域名后，Resend会提供几条DNS记录。

一般包括：

* SPF
* DKIM
* DMARC

在域名服务商后台添加即可，例如：

* Cloudflare
* Namesilo
* GoDaddy

添加完成后等待几分钟，Resend会自动验证。

---

# 创建 API Key

进入：

```
Dashboard → API Keys → Create API Key
```

这里有两个权限类型：

| 权限             | 使用场景            |
| -------------- | --------------- |
| Sending access | 仅发送邮件           |
| Full access    | 管理Audience、订阅者等 |

如果只是交易邮件，例如：

* 注册验证
* 支付收据

**Sending access 就足够。**

如果未来要做Newsletter，需要 **Full access**。

注意：

**API Key只显示一次，创建后需要立即保存。**

---

# Supabase 接入自定义 SMTP

这一部分是整个系统最关键的一步。

目的只有一个：

**解决邮件送达率问题。**

进入：

```
Supabase Dashboard
→ Authentication
→ SMTP Settings
```

开启：

```
Custom SMTP
```

填写以下参数。

| 配置项          | 参数                                                                |
| ------------ | ----------------------------------------------------------------- |
| Host         | smtp.resend.com                                                   |
| Port         | 465                                                               |
| Username     | resend                                                            |
| Password     | Resend API Key                                                    |
| Sender email | [noreply@mail.yourdomain.com](mailto:noreply@mail.yourdomain.com) |
| Sender name  | 产品名称                                                              |

保存之后：

Supabase的所有认证邮件都会通过你的域名发送，包括：

* 注册验证邮件
* Magic Link登录
* 密码重置邮件

**邮件送达率会明显提升。**

```javascript

  from: 'Product <noreply@mail.yourdomain>',
```

# 两个常见问题

在实际配置过程中，我遇到了两个典型问题。

## 域名未验证导致 403

例如：

我在Resend验证的域名是：

```
mail.fyyd.net
```

但代码里写的是：

```
hello@chuhai.fyyd.net
```

结果直接返回：

```
403
```

规则很简单：

* **@前面可以随意**
* **@后面的域名必须和Resend验证的域名完全一致**

正确示例：

```
noreply@mail.fyyd.net
support@mail.fyyd.net
hello@mail.fyyd.net
```

---

## Supabase默认SMTP Gmail收不到

这个问题非常常见。

原因是：

**共享SMTP信誉低。**

一旦配置自定义SMTP之后，这个问题基本就消失了。
