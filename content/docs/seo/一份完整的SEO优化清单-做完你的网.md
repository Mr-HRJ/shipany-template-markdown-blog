# 一、SEO基础清单

这一部分解决的是底层问题。基础没打好，后面很多优化动作都很难跑出效果。

## 1. 设置Google Search Console

Google Search Console是SEO最基础、也最重要的免费工具之一。只要你认真做SEO，这一步就不能跳。

它能帮我做几件非常关键的事：

* 看哪些关键词正在带来流量
* 提交网站地图
* 修复抓取和索引错误
* 查看Core Web Vitals表现

时间范围：**12/19/23 - 3/12/24**

| ---------------------- | ----- |
| 总点击量 Total clicks      | 1.05M |

| 平均排名 Average position  | 20.7  |

**如果你还没装GSC，先装它。** 很多SEO判断，最后都要回到这里看数据。

## 2. 安装Bing Webmaster Tools

很多人只盯着Google，但Bing不该被忽视。它虽然整体份额不如Google，但每天依然有**超过1亿活跃用户**，而且还能提供另一组搜索表现数据。

它的核心功能包括：

* 搜索表现分析
* URL检查与索引状态查看
* 提交网站地图
* 查询反向链接
* 做关键词研究

## 3. 设置Google Analytics 4

Google Analytics 4能回答一个更现实的问题：**用户来到你的网站后，实际是怎么行动的。**

我会重点看这些数据：

* Google带来了多少流量
* 哪些页面最能吸引用户
* 流量趋势是在涨还是在掉
* 还有哪些网站或搜索引擎在给我导流

## 4. 安装Yoast SEO

如果你的网站是WordPress或Shopify，Yoast SEO是很省事的一步，尤其适合刚开始做SEO的人。

它能处理的事情包括：

* 优化页面标题和描述
* 自动生成网站地图
* 设置robots.txt
* 给出页面SEO评分建议
* 扫描网站并创建优化数据库

它不是万能工具，但作为基础插件，能帮你把很多容易漏掉的基础动作补上。

## 5. 明确SEO项目的KPI

SEO做久了最怕一件事：做了很多事，但不知道到底有没有变好。

所以在开始之前，我一定会先把KPI定清楚。常见的SEO KPI包括：

* 自然流量增长
* 品牌曝光度
* 反向链接增长
* 关键词排名提升
* 转化率和销售额增长

**先定义目标，再做动作。** 否则你会很容易陷进“今天修这个、明天调那个”的忙碌假象里。

# 二、关键词研究清单

关键词研究不是找几个词填进文章里，而是搞清楚：**用户到底在搜什么，他们为什么搜，以及这些需求值不值得我做。**

## 8. 明确目标受众

在找关键词之前，我会先回答四个问题：

* 我的理想客户是谁
* 他们正在碰到什么问题
* 他们平时活跃在哪些平台
* 他们通常会用什么词来搜索

关键词研究如果脱离用户，很容易做成一堆看起来有搜索量、实际上没转化价值的词。

这是最简单也最容易上手的方法之一。

做法很直接：在Google输入一个核心词，先别按回车，看下拉框里系统给出的自动建议。这些词往往都是真实用户高频搜索的问题。

| cold brew french press  |

这类词特别适合拿来做内容拓展，因为它们通常更具体，也更接近用户真实需求。

# 三、页面内SEO清单

页面内SEO的重点，不是机械塞关键词，而是让搜索引擎和用户都能更快理解你的页面到底讲什么。

## 15. 在URL里包含关键词

把目标关键词放进URL，可以更直接地告诉搜索引擎页面主题是什么，同时也更利于用户理解。

### 示例

* 目标关键词：SEO清单
* URL：backlinko.com/seo-checklist

## 16. 使用短URL

研究显示，**短URL通常更容易获得更好的排名表现。**

原因很简单：

* 更清晰
* 更好理解
* 更容易分享
* 页面主题更聚焦

## 17. 在标题标签里前置关键词

标题是最重要的页面信号之一。如果关键词能靠前放，通常更有利于相关性识别。

### 示例

* 标题：Copywriting: The Definitive Guide

这里“copywriting”被放在最前面，页面主题非常明确。

## 18. 给标题加修饰词

一些修饰词能明显提高点击率，比如：

* Best
* Top
* 完整指南
* 2025最新
* Template

### 示例

* Keyword Research for SEO: The Definitive Guide + Template

好的标题不是只顾SEO，也要顾点击。

## 19. 在前150词里出现关键词

开头这部分非常重要。它会直接影响搜索引擎和用户对页面主题的初步判断。

### 示例

> This is a list of the 41 best SEO tools and software platforms on the planet. In fact, these tools have helped my site get 500k+ visitors per month.

这一类开场，既把主题说清了，也把内容价值说清了。

## 20. 在H1、H2、H3中使用关键词

标题层级里出现关键词，有助于搜索引擎理解页面结构和重点。

例如这篇清单里，“SEO清单”就出现在标题层里，这对排名理解是有帮助的。

## 21. 优化图片

图片SEO很多人不重视，但它其实很基础。

我通常会注意两件事：

* 文件名写清楚
* ALT标签写清楚

### 示例代码

```html
![菠菜煎蛋](spinach_omelette.png)
```

这样做不只是为了图片搜索，也是在补足页面语义信息。

## 22. 使用同义词和LSI关键词

不要为了SEO把同一个词来回堆。更好的做法是用一组相关表达，把主题讲完整。

例如：

* 启动博客
* 创建博客
* WordPress建站

冷萃咖啡相关词也可以扩展成：

* COLD WATER
* TEMPERATURE
* BEANS
* ICE
* GRIND

**搜索引擎现在看的是主题覆盖，不是机械重复。**

## 23. 使用外部链接

给权威站点做外链引用，可以提升内容可信度。

比如链接到：

* Wikipedia
* Google
* 行业权威研究报告
* 官方文档

这会让页面更像一篇完整、可信的内容，而不是关起门来自说自话。

## 24. 使用内部链接

每篇文章尽量链接到**2-5个相关页面**，这样做的价值很直接：

* 帮搜索引擎理解网站结构
* 传递页面权重
* 提升用户浏览深度

内部链接做得好，整个网站的主题网络会更清楚。

# 四、内容优化清单

内容优化的重点，不是把文章写长，而是让它更容易读、更有价值、更值得被引用。

## 25. 分段排版，提升可读性

如果一篇文章全是大段文字，用户很难看完，搜索引擎也不一定认为它体验好。

我会优先做这些排版动作：

* 多用小标题
* 控制段落长度
* 多用列表
* 把重点结论单独拎出来

## 26. 聚焦当前还有效的内容格式

不是所有内容格式都一样能打。现在更容易做出结果的，通常包括：

* 专家撰写内容
* 原创研究
* 案例研究
* 常青内容

这些内容更容易被引用，也更容易建立长期搜索资产。

## 27. 提供信息增益

现在做SEO内容，最怕的就是“换个说法重复别人说过的话”。

真正有效的内容，必须有**信息增益**。也就是：

* 独特观点
* 新数据
* 一手经验
* 更清晰的结构
* 更可执行的方法

**如果用户在你这里看到的内容，和搜索结果前十名几乎一模一样，那很难拿到明显优势。**

## 28. 使用多媒体

多媒体不只是为了好看，它对停留时间和理解效率都很有帮助。

常见形式包括：

* 图片
* 图表
* 视频
* 信息图

如果一段内容本来就适合可视化，就不要硬堆成纯文字。

# 五、技术SEO清单

技术SEO解决的是搜索引擎能不能顺利抓、顺利读、顺利收录，以及页面加载和使用体验够不够好。

## 29. 检查抓取和索引错误

先去Google Search Console的“索引”报告里看，哪些页面已经被收录，哪些没有，以及没收录的原因是什么。

### 索引数据示例

时间范围：**3/15/24 - 6/3/24**

| 指标      | 数值    |
| ------- | ----- |
| 已索引页面   | 21.3K |
| 未索引页面   | 495   |
| 未索引原因类型 | 9种    |

这类数据很关键，因为页面没被索引，再好的内容也拿不到搜索流量。

## 30. 查看Google是怎么查看你的页面的

GSC里的URL检查功能非常实用，我会用它核对这些问题：

* 页面是否已被Google收录
* HTTPS状态是否正常
* 结构化数据有没有问题
* canonical设置对不对
* 面包屑导航是否配置正常

很多页面排名异常，最后查下来都是这些基础设置出了问题。

## 31. 确保网站适配移动端

Google现在是移动优先索引，所以移动端体验不是加分项，而是基本项。

我会重点确认：

* 是否是响应式设计
* 是否有影响阅读的弹窗
* 句子和段落是否适合手机阅读

移动端做得差，排名和转化通常都会被一起拖累。

## 32. 修复断链

断链会影响用户体验，也会影响搜索引擎抓取效率。

可以用DrLinkCheck.com之类的工具去扫全站。

### 扫描示例

域名：**backlinko.com**

| 指标   | 数值    |
| ---- | ----- |
| 总链接数 | 1,500 |
| 无效链接 | 0     |

### 主要外部链接示例

| 目标域名               | 链接数 |
| ------------------ | --- |
| semrush.com        | 33  |
| support.google.com | 29  |

断链问题不一定大，但它属于典型的“该修就尽快修”的问题。

## 33. 使用HTTPS

HTTPS已经是Google明确的排名因素之一。如果网站还没全站启用HTTPS，这件事优先级很高。

## 34. 检查加载速度和Core Web Vitals

页面速度不只是技术问题，它直接影响用户体验和搜索表现。

我一般会用PageSpeed Insights去看这些指标。

### 核心指标示例

| 指标           | 数值    |
| ------------ | ----- |
| 最大内容绘制 LCP   | 1.6s  |
| 交互到下一次绘制 INP | 677ms |
| 累积布局偏移 CLS   | 0     |

### 优化建议

| 问题               | 当前情况 |
| ---------------- | ---- |
| 减少主线程工作          | 6.5s |
| 降低JavaScript执行时间 | 4.1s |
| 减少第三方代码影响        | 需要优化 |
| 优化图片格式和尺寸        | 需要优化 |

这些指标里，LCP、INP、CLS都值得持续盯着。尤其是JavaScript过重和第三方脚本过多，经常会拖慢整站。

## 35. 使用结构化数据 Schema

结构化数据的核心作用，是帮助搜索引擎更准确理解页面内容，并增加拿到富文本摘要的机会。

常用工具包括：

* Google官方Rich Results Test
* Schema Markup Validator

### 常见结构化数据类型

| 类型                   |
| -------------------- |
| 文章 Article           |
| 产品 Product           |
| 食谱 Recipe            |
| 评论摘要 Review snippets |
| 视频 Videos            |

如果你的网站内容本身适合加Schema，不做其实有点可惜。

# 六、外链建设清单

外链建设不是盲目求量，而是想办法拿到真正有价值、相关性高的链接。

## 36. 为媒体提供专家引用

可以通过这些平台向媒体提供专家观点：

* Connectively 原HARO
* Help a B2B Writer

这类方式的优势是，你提供的是专业观点，而不是硬做链接交换。

### 示例需求

| 主题             | 需要的观点类型 |
| -------------- | ------- |
| 建筑行业危险材料安全处理   | PPE相关评论 |
| 60/70年代葡萄酒鸡蛋饮食 | 专家解读    |
| 噪音污染对身心健康的影响   | 亲历者分享   |

如果回答得足够专业，被引用后往往能拿到不错的媒体外链。

## 37. 创建可链接资产

真正强的外链资产，通常不是普通文章，而是别人愿意主动引用的内容。

典型形式包括：

* 原创研究报告
* 数据图表
* 行业排名
* 可引用的统计页面

### 示例

Backlinko的“Google排名因素”页面，拿到了：

| 指标   | 数值   |
| ---- | ---- |
| 反向链接 | 26K+ |
| 引用域名 | 6.7K |

这类内容的外链增长，往往不是靠一封封发邮件，而是因为内容本身足够值得被引用。

## 38. 反查竞争对手外链

用Semrush的Backlink Analytics去看竞争对手的外链来源，是非常高效的做法。

你可以直接看：

* 哪些站在链接他们
* 哪些页面最容易吸引链接
* 他们的外链策略重心在哪

### 示例外链来源

| 来源站点                     | 类型     |
| ------------------------ | ------ |
| contenta.tw/archives/961 | 中文博客   |
| webdesignerdepot.com     | 英文技术博客 |
| forbes.com               | 大型媒体   |
| smallbizgenius.net       | 小企业指南站 |

这一步的本质不是“照抄”，而是看清对手是怎么拿链接的，然后找自己可复制的路径。

## 39. 成为播客嘉宾

播客是一个经常被低估的外链和品牌曝光渠道。

### 示例

| 项目   | 内容                  |
| ---- | ------------------- |
| 播客主题 | YouTube SEO最新趋势     |
| 嘉宾角色 | SEO专家分享关键词策略、视频优化技巧 |

参加合适的行业播客，通常能同时带来：

* 品牌曝光
* 个人背书
* 官网外链
* 后续合作机会

## 40. 在文章中提及影响者

如果你的文章里提到了行业里有影响力的人或品牌，可以在发布后通知对方。

这不一定能换来外链，但有机会换来：

* 转发
* 点赞
* 二次传播
* 后续合作

### 示例邮件模板

> 嘿 Mark 和 Anna，
> 新文章刚发布：[http://backlinko.com/seo-tools](http://backlinko.com/seo-tools)
> 文中特别推荐了 DeepCrawl，如果你喜欢这篇指南，欢迎转发、点赞或在inbound.org投票支持！
> 谢谢，
> Brian

这类动作的关键不是群发，而是你文章里真的给了对方有价值的提及。

# 七、AI SEO清单

现在的SEO已经不只是Google十个蓝色链接，还包括AI搜索、AI答案和品牌在生成式结果里的可见性。

## 41. 明确受众在用哪些AI工具

不要上来就什么都优化。先确认你的受众到底主要在哪些AI工具里提问。

常见工具包括：

* ChatGPT
* Perplexity
* Google AI Mode

我的建议是：**先抓前2-3个最关键的平台。**

## 42. 收集真实提示词 Prompts

如果你想让内容更容易被AI匹配，就要知道用户到底怎么问。

提示词可以从这些地方收集：

* 客服记录
* Reddit
* Quora
* 销售沟通记录
* 用户访谈

### 示例提示词

| Prompt                                                                   | 核心需求         |
| ------------------------------------------------------------------------ | ------------ |
| Which tools are most effective for tech content creation?                | 技术内容创作工具推荐   |
| Can you recommend tools to analyze and identify micro niches?            | 微niche分析工具推荐 |
| What are the best platforms for content creators to showcase their work? | 内容创作者展示平台推荐  |

这些真实问法，往往比你自己想出来的SEO关键词更接近真实搜索场景。

## 43. 测试品牌AI可见性

现在已经有工具能追踪品牌在AI结果里的可见性，比如Semrush AI Visibility Toolkit。

### 数据示例

时间范围：**Sep 2025 - Oct 2025**

| 指标      | 数值             |
| ------- | -------------- |
| AI可见性得分 | 36/100 中等      |
| 月受众     | 21.2M 下降229.5K |
| 提及次数    | 731 下降131      |
| 被引用页面   | 2.1K 下降441     |

如果你的品牌在AI答案里长期不出现，或者出现得很少，那说明上游内容和品牌信号还需要加强。

## 44. 记录AI引用的来源

AI并不是凭空生成答案，它通常会反复依赖一些来源站点。

常见引用来源可能包括：

* Reddit
* YouTube
* 评测站
* 行业博客

### 示例引用来源

| 来源                              |
| ------------------------------- |
| HubSpot Blog                    |
| Backlinko                       |
| Content Marketing Institute CMI |
| Animalz Blog                    |

这一步很有价值，因为它能告诉我：**如果我想提高被AI引用的概率，我应该优先在哪些内容生态里建立存在感。**

## 45. 创建AI友好回答页

如果一个问题用户会反复问，那就可以专门做一个结构清晰、回答直接、方便引用的页面。

这类页面的特点通常是：

* 问题明确
* 答案结构清楚
* 语义集中
* 少废话
* 方便提取

现在很多AI结果，偏爱这种“拿来就能引用”的内容结构。

## 46. 监控品牌在AI回答中的情感倾向

除了有没有被提到，还要看AI是怎么描述你的品牌的。

重点关注这些情况：

* 是否带有负面倾向
* 信息是否过时
* 是否遗漏关键优势
* 是否把你和竞品的定位说混了

### 情感分析示例

| 品牌        | 情感区间    |
| --------- | ------- |
| Backlinko | 80%-94% |
| Ahrefs    | 23%-47% |
| Semrush   | 21%-40% |

### 总体情感分布示例

| 情感类型 | 占比  |
| ---- | --- |
| 正面情感 | 85% |
| 中性情感 | 15% |

如果AI对品牌的描述长期不准确，就要回头修正官网内容、权威站点信息和外部品牌信号。

# 八、高级SEO技巧

这一部分更像是放大器。前面基础动作做好后，这些技巧能帮你进一步把流量做上去。

## 47. 优化用户体验信号

SEO最后拼的，不只是抓取和关键词，还包括用户是否愿意点、愿意看、愿意继续浏览。

我会重点优化两类信号。

### 提升点击率

方式包括：

* 优化标题
* 优化描述
* 使用结构化数据
* 让搜索摘要更有吸引力

### 提升停留时间

方式包括：

* 提供更高质量的内容
* 图文结合
* 做清晰的内部链接
* 让页面结构更好读

**排名拿到只是开始，用户愿不愿意继续看，才决定这页能不能稳住。**

## 48. 删除死重页面

很多网站流量上不去，不只是因为内容不够多，也可能是因为垃圾页面太多。

常见“死重”页面包括：

* 空分类页
* 没流量的产品页
* 重复内容页
* 薄内容页面

这些页面会稀释整体质量信号，也会拖累抓取预算。

所以别只想着加页面，有时候**删页面、合并页面、清理页面**反而更重要。

## 49. 更新并重新发布旧内容

这是我特别认可的一招，因为它往往投入不算大，但回报很夸张。

你不一定非要一直写新文章。很多时候，把旧内容重新更新、补图、补数据、重构排版，再发布一次，效果会非常明显。

### 示例

| 指标    | 数值                                    |
| ----- | ------------------------------------- |
| 文章标题  | SEO COPYWRITING: The Definitive Guide |
| 更新时间  | 2023年11月7日                            |
| 更新后流量 | 6,433                                 |
| 更新前流量 | 535                                   |
| 增长率   | 1,102.43%                             |

这说明一件事：**旧内容不是资产包袱，而是可反复增值的流量资产。**

# 结论

如果把这份清单压缩成一句话，那就是：

**SEO增长，靠的不是一个技巧，而是一整套从基础到放大的系统执行。**

真正该做的顺序通常是：

1. 先把基础工具和数据体系搭起来
2. 再做关键词和主题研究
3. 接着优化页面结构和内容质量
4. 然后修技术问题
5. 最后通过外链、AI可见性和旧内容更新持续放大结果

如果你现在就要开始动手，我建议按这个顺序推进：

## 建议的执行顺序

1. 先装好GSC、GA4、Bing Webmaster Tools
2. 明确你的SEO KPI
3. 找出一批低竞争、高相关关键词
4. 优先优化已有页面，而不是急着大量发新内容
5. 修复索引、断链、速度和移动端问题
6. 开始布局外链和可链接资产
7. 同时把AI搜索可见性纳入监控
8. 定期更新旧内容，放大已有流量资产

**不要想着一天把49项全做完。**
真正靠谱的做法，是每周推进一部分，把最关键的动作先落地。
